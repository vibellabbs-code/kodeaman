<Info>
  **AspidaSec exposes no public HTTP REST API.** Its programmatic surface is the **MCP server** (for AI assistants) and the **CLI** (for humans and CI). This section documents the MCP tools. Because there is no REST service, there is no OpenAPI specification to publish — generating one would mean inventing endpoints that do not exist, which the [trust model](/security-analysis/trust-model) forbids.
</Info>

## What the MCP server is

The package `@aspidasec/mcp-server` ships an executable, `aspidasec-mcp`, that starts a **stdio** MCP server named `aspidasec`. It lets AI coding assistants run scans, inspect findings, and request remediation without leaving the assistant.

## Prerequisites

- Node.js 20 or newer.
- AspidaSec installed from npm or available from the repository.
- Scanner dependencies for the checks you want to run.
- **Absolute paths.** Tools that take `repoRoot` require an absolute path, not a relative one.

## Conventions

| Convention | Detail |
| --- | --- |
| Transport | stdio |
| `language` | Optional enum `en` \| `id` on most tools. |
| `repoRoot` | Absolute path string, required where listed. |
| Stringified args | `finding` / `findings` / `scannerCoverage` are passed as JSON **strings**, not objects. |
| Default return | JSON, unless `format` selects `markdown` or `sarif`. |
